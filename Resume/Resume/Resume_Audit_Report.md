## 3. Audited & Resolved Issues

### **Issue 1: Exaggerated Scope of Responsibility ("Architected")**
* **Class:** Engineering Manager / Hiring Manager Critique (Moderate Risk)
* **Finding:** Describing a Junior Software Engineer (with 2.5+ years of experience) as having "architected" production applications from scratch (such as *Shojja Hospital*) is a potential red flag that senior managers can easily spot during interviews.
* **Resolution:** Replaced all instances of "Architected" or "Architect and implement" with grounded engineering terms:
  * *ATS Resume:* Changed to *"Developed core modules and implemented Clean Architecture with Riverpod..."*
  * *Recruiter Resume:* Changed to *"Developed features and implemented Clean Architecture with Riverpod..."*
  * *One-Page Resume:* Changed to *"Co-developed a hospital ecosystem app using Clean Architecture and Riverpod..."*

### **Issue 2: Claims of Workspace Optimization scale ("Delivered from scratch")**
* **Class:** Senior Recruiter Critique (Low Risk)
* **Finding:** Saying you delivered a multi-tenant client "from scratch" implies sole development. On a team of 4, this is unlikely.
* **Resolution:** Rephrased the impact statement to: *"Co-developed a secure, multi-tenant mobile client..."* to reflect team collaboration and accurate project scope.

### **Issue 3: Unverified Load and Metric Parameters ("Under continuous load" / "Replaced manual workflows")**
* **Class:** Engineering Manager Critique (Low Risk)
* **Finding:** Assertions like "under continuous load" or "replaced manual workflows" cannot be quantitatively backed up by project metrics in the workspace files.
* **Resolution:** 
  * Rephrased *Weighing Scale App* to: *"...stream, process, and display weight measurements."* (Removed *"under continuous load"*).
  * Changed impact to: *"Facilitated real-time weight tracking for industrial operators..."* (Removed *"replaced manual logging workflows"*).

### **Issue 4: Redundant Keywords & Technical Overlap**
* **Class:** ATS Parser Critique (Low Risk)
* **Finding:** Listing too many versions of the same tool (e.g. Git, GitHub, Git Version Control) in different sections wastes space.
* **Resolution:** Streamlined tools in the skills sections to avoid clutter while retaining target terms for parser matching.
